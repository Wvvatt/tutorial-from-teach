4. EME API 骨架

const access = await navigator.requestMediaKeySystemAccess(
  "com.widevine.alpha",
  [{
    initDataTypes: ["cenc"],
    videoCapabilities: [{
      contentType: 'video/mp4; codecs="avc1.640028"',
      robustness: "SW_SECURE_DECODE"
    }]
  }]
);

const mediaKeys = await access.createMediaKeys();
await video.setMediaKeys(mediaKeys);
const session = mediaKeys.createSession();

session.addEventListener("message", async event => {
  const license = await fetch(licenseServer, {
    method: "POST",
    body: event.message
  }).then(r => r.arrayBuffer());
  await session.update(license);
});

await session.generateRequest("cenc", initData);

5. Widevine L1 vs L3

维度L1L3
密钥位置TEE / secure hardware用户态 CDM 内存
解密位置TEE 或硬件安全路径软件解密
解码后帧secure buffer / secure dma-buf普通 YUV / GPU buffer
显示secure overlay + HDCP普通合成路径
典型平台Android TV、Chromebook、认证设备Desktop Linux/Chrome 常见

6. L3 软件路径

Encrypted segment
  -> MSE / demuxer
  -> encrypted DecoderBuffer
  -> CDM process decrypts clear bitstream
  -> software or hardware decoder
  -> normal VideoFrame
  -> GPU compositor / overlay if allowed

L3 的密钥和明文帧对 OS 安全边界来说更可见，因此内容策略通常限制分辨率或输出能力。

7. L1 secure path

Encrypted bitstream
  -> CDM / TEE key handle
  -> secure decrypt
  -> hardware decoder
  -> secure dma-buf
  -> secure overlay plane
  -> HDCP-protected connector

L1 的关键不是“能不能播”，而是明文 key 和解密后的像素是否永远不暴露给普通世界。

8. TEE 与 secure dma-buf

TEE 通过 TrustZone/OP-TEE 等机制把 secure world 与 Linux normal world 隔离。secure dma-buf 可以作为句柄出现在普通世界，但 CPU map 和非 secure GPU shader 读取应被拒绝。

if (dma_buf_is_secure(dbuf) && request_cpu_map)
  return -EPERM;

if (fb->secure && !plane_supports_secure_scanout)
  return -EACCES;

9. 输出保护

HDMI/DP 外接显示还需要 HDCP。Linux DRM connector 的 Content Protection property 常见状态包括 UNDESIRED、DESIRED、ENABLED。
