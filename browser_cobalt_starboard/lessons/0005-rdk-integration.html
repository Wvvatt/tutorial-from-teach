Lesson 0005: RDK Integration — 平台服务与焦点管理

嵌入式媒体平台开发 · 第五课

之前的课程聚焦在内部实现。这一课看向外部：AML Cobalt 如何与 RDK 平台（WPEFramework/Thunder）通信。这是让 Cobalt 融入更大系统的关键——焦点管理、TTS、设备信息都来自这里。

1. RDK 与 WPEFramework

RDK (Reference Design Kit) 是有线电视/IPTV 行业的参考平台。它的核心通信框架是 WPEFramework (Thunder)，提供基于 JSON-RPC 的进程间通信。每个功能是一个 plugin，通过 callsign 标识。

--- 关键 callsign ---

org.rdk.RDKShell.1       ← 窗口/焦点管理
org.rdk.TextToSpeech.1   ← 语音合成

ServiceLink 类封装了与 Thunder 的 JSON-RPC 通信：

--- rdkservices.cc:62-138 ---

class ServiceLink {
  JSONRPC::LinkType<...> link_;

  // Thunder → Get (同步请求)
  template<typename PARAMS>
  uint32_t Get(waitTime, method, params);

  // Thunder → Dispatch (异步请求 + 回调)
  template<typename HANDLER>
  uint32_t Dispatch(waitTime, method, callback, object);

  // Subscribe (订阅事件)
  template<typename INBOUND>
  uint32_t Subscribe(waitTime, eventName, method, object);

  // 此外：环境变量重写 (COBALT_ENABLE_OVERRIDES)
  // 如果设置了，可以从环境变量读取 JSON 响应
  // 用于测试 —— 无需真实 Thunder 即可模拟
};

2. 焦点管理 — RDKShell

在 Lesson 1 中我们看到 Application 持有 amlshellinfo_ 并在 GetFocusStatus/GetFocusAppName 中使用它。这里就是其实现：

--- rdkservices.cc:278-363 ---

struct RDKShellInfoImpl {
  ServiceLink rdkshellinfo_link_ { "org.rdk.RDKShell.1" };
  bool   focus_status_ {false};
  string appName_;          // 自身 app 名 (来自 CLIENT_IDENTIFIER)
  string focusAppName_;     // 当前焦点 app

  RDKShellInfoImpl() {
    // 1. 从环境变量获取自身 app 名
    appName_ = getenv("CLIENT_IDENTIFIER");  // 转为小写

    // 2. 查询当前焦点
    Get("getFocused") → 如果自己就是焦点 → SendFocusEvent()

    // 3. 订阅焦点变化事件
    Subscribe("onApplicationFocusChanged", onFocusStatus);
  }

  void onFocusStatus(JsonObject data) {
    if (焦点转移到自己 && !focus_status_)
      → SbEventSchedule(SendFocusEvent, 0)       // 获得焦点
    else if (焦点转移走 && focus_status_)
      → SbEventSchedule(SendBlurEvent, 0)        // 失去焦点
  }
};

关键时序：App 启动时先查询当前焦点状态，然后订阅后续变化事件。

🔑 事件注入模式
SbEventSchedule(callback, 0) 被用来将焦点事件注入到 Cobalt 主线程的事件队列中。这是一个典型的线程切换模式——RDK 的回调在 Thunder 的线程中触发，通过 SbEventSchedule 安全地切换到 Application 的 Starboard 事件循环。

3. 焦点在 Application 中的使用

RDKShellInfo 的焦点状态在 Application 中被用于两个关键地方：

--- application_aml.cc:388-393 ---

bool Application::GetFocusStatus() const {
  return amlshellinfo_->ImpGetFocusStatus();
}
std::string Application::GetFocusAppName() const {
  return amlshellinfo_->ImpGetFocusAppName();
}

这些被 HdcpService 使用：

--- aml_platform.cc:396-424 ---

SetFreeze(freeze) {
  if (不冻结 && (!GetFocusStatus() && appName != "launcher")) {
    跳过 HDCP 处理  // 不是焦点 app 且不是 launcher → 不处理
  }
  if (不冻结)
    → SendUnfreezeEvent + SendRevealEvent + SendFocusEvent
  else
    → SendBlurEvent + SendConcealEvent + SendFreezeEvent
}

这就是完整的 HDCP 保护链路：

HDMI 拔出
  → udev event
    → HdcpService::onEvent()
      → SetFreeze(true)
        → SendBlurEvent + SendConcealEvent + SendFreezeEvent

HDMI 插入 + HDCP 验证通过
  → udev event
    → HdcpService::onEvent()
      → SetFreeze(false)
        → SendUnfreezeEvent + SendRevealEvent + SendFocusEvent

4. TTS — Text-to-Speech

TextToSpeech 是一个独立的 RDK 服务，通过 Thunder 通信：

--- rdkservices.cc:140-276 ---

struct TextToSpeechImpl {
  ServiceLink tts_link_ { "org.rdk.TextToSpeech.1" };
  atomic_bool is_enabled_;

  TextToSpeechImpl() {
    Subscribe("onttsstatechanged", OnStateChanged);
    Get("isttsenabled") → 设置 is_enabled_
  }

  void Speak(text) {
    Dispatch("speak", params, OnSpeakResult);
  }

  void Cancel() {
    Dispatch("cancel", params, OnCancelResult);
  }
};

在 Starboard 层被 speech_synthesis_speak.cc 调用：

--- speech_synthesis_speak.cc ---

void SbSpeechSynthesisSpeak(const char* text) {
  TextToSpeech::Speak(text);
}

同样，speech_synthesis_cancel.cc 和 accessibility_get_text_to_speech_settings.cc 也通过 TextToSpeech 实现。

5. ServiceLink 的两个隐藏特性

5.1 环境变量重写

--- rdkservices.cc:66-103 ---

enableEnvOverrides():
  如果 COBALT_ENABLE_OVERRIDES=1
  Get(method) 时，先检查环境变量 "{Callsign}_{method}"
  如果存在，直接解析环境变量 JSON 作为返回

  例如: org_rdk_RDKShell_1_getFocused={"client": "cobalt"}

这允许在 没有真实 Thunder 的环境 中调试——非常实用的嵌入式开发技巧。

5.2 THUNDER_ACCESS 环境变量

ServiceLink(callsign) {
  if (getenv("THUNDER_ACCESS") != nullptr)
    link_ = new JSONRPC::Link(callsign);
  // 否则 link_ 为空，所有操作返回 UNAVAILABLE
}

如果没有设置 THUNDER_ACCESS，Cobalt 可以运行但所有 RDK 服务都不可用。
