Lesson 0008: ESS Input — 按键/鼠标/遥控器处理

嵌入式媒体平台开发 · 第八课

之前的课程都是内部处理。这一课看外部输入到内部事件的转换——ESSOS 的按键/指针回调如何变成 Starboard SbInputData 事件注入 Cobalt。

1. 事件流全景

--- ess_input.cc:547-623, application_aml.cc:62-69 ---

ESSOS 驱动层
    │  keyListener / pointerListener 回调
    ▼
Application::OnKeyPressed/Released/OnPointerMotion/ButtonPress/Release
    │
    ▼
EssInput::OnKeyPressed/OnKeyReleased/OnPointerMotion/...
    │
    ├── KeyCodeToSbKey()         ← Linux event code → SbKey 映射
    ├── KeyCodeToSbKeyLocation() ← 按键位置 (左/右 Ctrl/Shift/Alt)
    ├── UpdateModifiers()        ← 跟踪 Ctrl/Shift/Alt/Meta 状态
    └── CreateKey()              ← 构造 SbInputData, 处理按键重复
         │
         ▼
    Application::Get()->InjectInputEvent(data)   ← 注入 Starboard 事件队列

2. Key Code 映射表

KeyCodeToSbKey() 是一个大 switch-case，映射 Linux linux/input.h 的 event code 到 Starboard 的 SbKey：

--- ess_input.cc:56-370 ---

特殊映射:
  KEY_F12     → kSbKeyMicrophone    ← 语音键
  KEY_PLAY    → kSbKeyPlay
  KEY_F1      → kSbKeyLaunchThisApplication

  KEY_BACK    → kSbKeyBrowserBack   ← USB 键盘多媒体键
  KEY_MUTE    → kSbKeyVolumeMute
  KEY_VOLUMEUP/DOWN → kSbKeyVolumeUp/Down
  KEY_CHANNELUP/DOWN → kSbKeyChannelUp/Down

  KEY_REWIND/FASTFORWARD → kSbKeyMediaRewind/FastForward
  KEY_PLAYPAUSE → kSbKeyMediaPlayPause

  KEY_RED/GREEN/YELLOW/BLUE → kSbKeyRed/Green/Yellow/Blue
  KEY_SUBTITLE  → kSbKeySubtitle
  KEY_AUDIO     → kSbKeyMediaAudioTrack
🔑 行277-288 的注释揭示了嵌入式开发的真实痛点
KEY_F5（红外码 116）被故意不映射——因为电视关机时 CEC 会发 F5 键，导致 Cobalt 重新加载页面并使进程挂起。注释说"这应该是测试页面的问题"，但为了稳定性直接禁掉了。

电视遥控器按键少，Ctrl 组合键提供了额外的功能：

CreateKey() 如果 modifiers == Ctrl:

4. 按键重复逻辑

YouTube 技术规范要求：按键按下 500ms 后每隔 50ms 重复发一次 keydown：

--- ess_input.cc:46-53, 485-506 ---

kKeyHoldTime = 500ms, kKeyRepeatTime = 50ms

CreateKey(key, Press, repeatable):
  1. 构造 SbInputData → InjectInputEvent
  2. 如果 repeatable && Press:
     key_repeat_key_ = key
     SbEventSchedule(CreateRepeatKey, this, kKeyHoldTime)
     // 500ms 后触发 ⬇

CreateRepeatKey():
  1. key_repeat_interval_ 从 500ms 改为 50ms
  2. 递归调用 CreateKey(Press, repeatable)
     // 又一次 Inject + Schedule → 50ms 后再次触发
     // 循环直到用户松手 → DeleteRepeatKey()
🔑 设计巧妙处
关键逻辑在第 537-538 行: if (Press && repeatable && key == key_repeat_key_ && key_repeat_state_) return;
如果按键按下期间又收到同键的 Press 事件（ESSOS 底层重复上报），直接跳过——防止重复的重复。只有 Release 事件才能终止循环。

5. Modifier 跟踪

--- ess_input.cc:517-527 ---

UpdateModifiers(key, type):
  if (key 是 Ctrl/Shift/Alt/Meta):
    Press   → key_modifiers_ |= modifier
    Unpress → key_modifiers_ &= ~modifier
    return true  ← 不产生输入事件（修饰键本身不是输入）

  只有普通键在有 modifier 时才会处理（如 Ctrl+M）

6. 指针/鼠标支持

--- ess_input.cc:555-623 ---

OnPointerMotion(x, y):           → SbInputEventTypeMove
OnPointerButtonPress(btn, x, y): → SbInputEventTypePress, kSbInputDeviceTypeMouse
OnPointerButtonRelease(btn, x, y): → SbInputEventTypeUnpress

鼠标按钮映射: BTN_LEFT → kSbKeyMouse1, BTN_RIGHT → kSbKeyMouse2

指针和键盘的区别：指针事件设置 data->position.x/y，且 device_type 为 kSbInputDeviceTypeMouse 而非 Remote。
