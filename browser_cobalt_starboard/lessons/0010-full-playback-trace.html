Lesson 0010: 一次性播放请求的完整链路

嵌入式媒体平台开发 · 第十课 (综合)

从用户点击 YouTube 视频到画面出现在电视上，Cobalt 走了哪些路径？这一课以一次 H264 + AAC + Widevine DRM 的加密视频为例，完整追踪。

HDR? codec∈{H264,VP9,AV1}? → Application::GetDisplayResolution() → DeviceInfoService (L1) (3) SbMediaIsAudioSupported() → media_is_audio_supported.cc:59 → 检查: Dolby 支持? bitrate? GstRegistryHasElementForMediaType

阶段四: 数据推送与解密

(11) Cobalt 发送第一个 Seek(0) → Pipeline 进入 PAUSED 状态 → state_ = kInitialPreroll → ChangePipelineState(GST_STATE_PAUSED) ← 开始 buffering → 通知 Cobalt: kSbPlayerStatePrerolling → Pipeline ASYNC_DONE → state_ = kPresenting (12) GStreamer appsrc need-data 回调触发: → AppSrcNeedData() → DecoderNeedsData(MediaType::kVideo|kAudio) → Cobalt 收到 kSbPlayerDecoderStateNeedsData → 开始推送样本 (13) SbPlayerWriteSample2(player, kSbMediaTypeVideo, sample_infos, 1) → player_write_sample.cc:30 → player_->WriteSample(sample_type, sample_infos, count) (14) PlayerImpl::WriteSample(SbPlayerSampleInfo*) (player_internal.cc:1847) → 分配 GstBuffer, 拷贝 Cobalt 数据 → 解析 drm_info: identifier(key_id), iv, subsamples → SessionIdByKeyId(key_id) → session_id → DRM key 未就绪? → PendingSample 挂起 DRM key 就绪? → 继续 ↓ (15) PlayerImpl::WriteSample(GstBuffer*) (player_internal.cc:1609) → [secure = true] DRM + video: ├─ gst_buffer_new_allocate(allocator_, size) → buffer2 (secmem) ├─ gst_secmem_memory_get_handle(mem) → handle └─ gst_buffer_add_protection_meta(buffer, drm_info) ← handle 挂原始 buffer (16) DrmSystemWidevine::Decrypt(GstBuffer*) (drm_system_widevine.cc:956) → DecryptImpl(...) → DecryViaWv(id, mediaType, subsamples, iv, data, keyId, ..., sec_handle) → 构造 wv3cdm::DecryptionBatch → [video]: is_secure=true, output.data = sec_handle ← 硬件解密写入 secmem → [audio]: is_secure=false, 分配临时缓冲区 → cdm_->decrypt(decryptionBatch) ← Widevine CDM → TrustZone 硬件解密 (17) 解密后处理: → [VP9/AV1]: gst_secmem_parse_vp9/av1(mem) → gst_app_src_push_buffer(GST_APP_SRC(src), buffer2) ↑ 此时 buffer2 中的数据在 secmem 中, CPU 不可读, 只有硬件解码器可访问 (18) 缓冲水位管理: → 每 80ms: CheckVideoBufferHealth() 巡检 → video_queued < 250ms? → internal pause (pipeline_is_paused_internal_) → video_queued > 2000ms && audio > 900ms? → resume PLAYING

阶段五: GStreamer 管线处理

(19) GStreamer pipeline 处理加密样本: [视频路径] appsrc my_vsrc → h264secparse (安全解析器, 提取 NAL units) → [硬件视频解码器] (从 secmem 读取, 解码 H264 → YUV) → westerossink / amlvideosink → Westeros Wayland compositor → DMA-BUF 传输 → DRM/KMS → GPU overlay → HDMI 输出 [音频路径] appsrc my_asrc → aacparse (解析 AAC frames) → avdec_aac (软件解码 AAC → PCM) → audioconvert + audioresample (格式转换) → amlhalasink (Amlogic 硬件音频 sink) → ALSA → I2S/TDM → HDMI 音频输出 (20) 音视频同步: WESTEROS_GL_USE_AMLOGIC_AVSYNC=1 → Westeros 将视频 PTS 传给 Amlogic 硬件 AV 同步引擎 → 硬件控制 vsync 时机, 实现音频跟随视频

阶段八: 播放结束与清理

(27) EOS 到达: → SbPlayerWriteEndOfStream(type) → gst_app_src_end_of_stream(src) → 所有 stream EOS → pipeline 发送 GST_MESSAGE_EOS → BusMessageCallback → Dispatch: kSbPlayerStateEndOfStream → Cobalt 停止写入, UI 显示播放完成 (28) Player 销毁: → SbPlayerDestroy(player) → PlayerImpl::~PlayerImpl() → GetPlayerRegistry()->Remove(this) → 清理 GStreamer 资源 (source destroy, bus unwatch) → ChangePipelineState(GST_STATE_NULL) → 发送 PlayerDestroyedTask → g_main_loop_quit() → SbThreadJoin(playback_thread) → 释放 secmem allocator, DRM observer 注销
