AML Starboard 术语表

Starboard 核心概念

Starboard CORE
Cobalt 的平台抽象层 (Porting Layer / OS Abstraction)。封装了Cobalt所需的所有平台特定功能。每个平台需要实现 Starboard API 才能运行 Cobalt。
NPLB CORE
"No Platform Left Behind" — Starboard 的平台验证测试套件。每个移植必须通过 NPLB 测试以确保 API 实现正确。`starboard/nplb/`
Starboard ABI (SABI) CORE
Starboard 二进制接口规范。JSON 格式文件定义各架构的 API 版本、类型大小、字节序、指针宽度。确保 Evergreen 架构下 `libcobalt.so` 与 `loader_app` 的二进制兼容性。`starboard/doc/starboard_abi.md`
Evergreen CORE
Cobalt 的云端可更新架构。将 Cobalt Core (`libcobalt.so`) 与 Starboard 平台实现分离，允许通过 Google 更新服务推送 Cobalt 更新而不需要平台固件 OTA。`starboard/doc/evergreen/`
SB_API_VERSION CORE
定义平台实现的 Starboard API 版本的宏。写在平台 `configuration.h` 中。AML 当前主线基于 SB API 23，sb17/sb18 分支在新版本迁移中。

Amlogic 平台特定

AML / Amlogic AML
Amlogic（晶晨半导体）—— 领先的 SoC 厂商，专注智能电视、机顶盒、智能家居等嵌入式多媒体设备。
ESSOS AML
Amlogic 的图形和输入系统 (ESS OS)。提供 `EssCtx` 上下文、键盘/指针事件监听、窗口管理。AML Starboard 的 `Application` 类封装了 ESSOS 的事件循环。``
Yocto AML
AML 平台的构建系统基础。AML 使用 Yocto 项目来构建嵌入式 Linux 发行版。绝大多数 AML commit 标题以 "Yocto:" 开头。
RDK AML
Reference Design Kit — 有线电视/IPTV 行业的参考设计套件。AML 支持 RDK 平台，有独立分支 `23.lts.amlogic.rdk`。RDK 版本使用 GStreamer 媒体管线。
Westeros AML
Wayland 协议兼容的显示合成器，RDK 平台使用。AML 在 RDK 启动器中设置 `WESTEROS_GL_USE_AMLOGIC_AVSYNC` 和 `WESTEROS_SINK_AMLOGIC_USE_DMABUF` 环境变量以启用 AML 硬件同步。`starboard/rdk/launcher.py`
amlhalasink AML
Amlogic 硬件视频解码的 GStreamer element。RDK 移植中通过 `wait-video`、`a-wait-timeout`、`disable-xrun` 属性配置解码行为。`starboard/contrib/rdk/.../player_internal.cc`
libamlv4l2 AML
Amlogic 的 V4L2 封装库。用于硬件视频编码器 (`/dev/amvenc_avc`)。通过 `dlopen` 动态加载。`~/Code/cobalt/media/gpu/v4l2/aml_v4l2_device.cc`

媒体管线

Player MEDIA
Starboard 中控制媒体播放的 API。包括创建、销毁、seek、写入样本、设置音量、播放速率等。AML 实现在 `player/` 目录下有 15 个文件。`third_party/starboard/aml/shared/player/`
Media MEDIA
Starboard 中查询媒体能力的 API。包括编解码器支持检测、缓冲区配置、输出保护等。AML 实现在 `media/` 目录下有 19 个文件。`third_party/starboard/aml/shared/media/`
GStreamer MEDIA
开源多媒体框架。AML 的 RDK 移植使用 GStreamer 构建媒体管线。`gst_media_utils.cc` 是 AML GStreamer 辅助函数的实现。`third_party/starboard/aml/shared/media/gst_media_utils.cc`
Widevine MEDIA
Google 的 DRM 内容保护方案。AML 实现了 Widevine DRM 系统，支持 L1/L3 安全级别、会话管理、密钥存储、许可证解析。`third_party/starboard/aml/shared/widevine/`
PIP (Picture-in-Picture) MEDIA
画中画功能，在一个视频窗口之上叠加另一个小窗口。AML 通过 `max_video_capabilities` 参数检测 PIP 需求：当描述为 `"width=432; height=240; framerate=15;"` 时创建 PIP 播放器。PIP 模式下丢弃音频（`audio_codec_ = kSbMediaAudioCodecNone`），sink 名称使用 `-pip` 后缀，设置 `pip=TRUE` 和 `res-usage=0` 降低资源优先级。主+PIP 双路播放由 number_of_players_ >= 2 的限制保证。`third_party/starboard/aml/shared/player/player_internal.cc:999-1007`
Dual Video MEDIA
双视频并发解码。用于主视频 + PIP 画中画场景。AML 支持两个 `SbPlayer` 实例同时运行，每个实例有独立的 GStreamer pipeline 和 playback_thread。第二个播放器（PIP）降级为低分辨率（432x240@15）、纯视频模式。
secmem MEDIA
安全内存 (Secure Memory)，用于保护 DRM 解密后的视频数据。AML VP9/AV1 解码中使用 secmem 解析器替代插件方案。
Decode-to-Texture MEDIA
将解码后的视频帧直接渲染到纹理，用于 360° 球形视频等场景。AML 的 `player_get_current_frame.cc` 实现了 `SbPlayerGetCurrentFrame`。`starboard/doc/howto_decode_to_texture.md`

系统与构建

GN SYS
Generate Ninja — Cobalt/Satrboard 的构建系统（替代 GYP）。`BUILD.gn` 定义构建目标。AML 也遵循 GN 规范。`starboard/build/doc/gn_migrate_stub_to_platform.md`
platform_configuration SYS
GN 构建中描述平台特性的配置。每个二进制变体必须有 `platform_configuration/BUILD.gn` 和 `platform_configuration/configuration.gni`。AML 的 arm/arm64 各有自己的配置。`third_party/starboard/aml/arm/platform_configuration/`
Amlogic::Platform SYS
AML 平台服务框架。提供 `PlatformEvent` 事件总线和 `Subscriber` 接口。`DeviceInfoService`、`NetworkService`、`VoiceInputService`、`HDCP`、`CEC` 等都订阅此事件系统。`aml_platform.h`
CEC SYS
Consumer Electronics Control — HDMI CEC 协议。允许通过 HDMI 远程控制设备。AML 实现了 `CecEventHandle` 来响应 CEC 事件。`aml_platform.h`
HDCP SYS
High-bandwidth Digital Content Protection — HDMI 内容保护。AML 的 `HdcpService` 监控 HDMI 热插拔和 HDCP 状态（非 TV 平台）。`aml_platform.h`
PCM SYS
Pulse Code Modulation — 音频数据格式。AML 音频实现涉及 ALSA (Advanced Linux Sound Architecture) 接口。
