Demuxer 层

Demuxer 把容器格式拆成按轨道组织的 encoded frames。浏览器中最重要的分叉是 progressive 的拉模型与 MSE 的推模型。

1. 容器不是编码

MP4、WebM、MPEG-TS 是容器；H.264、VP9、AV1、AAC 是 codec。Demuxer 的工作是从容器中拆出音视频轨道、时间戳、codec config 和 encoded frame。

2. FFmpegDemuxer

DataSource(url/file)
  -> avformat_open_input()
  -> avformat_find_stream_info()
  -> create DemuxerStream per track
  -> av_read_frame()
  -> DecoderBuffer(audio/video)

Progressive 路径适合直接播放单个文件。seek 依赖容器索引和 byte range，demuxer 要找到目标时间之前最近的关键帧。

3. ChunkDemuxer

JS: sourceBuffer.appendBuffer(segment)
  -> SourceBuffer::AppendBuffer()
  -> ChunkDemuxer::AppendData()
  -> MP4StreamParser / WebMStreamParser
  -> FrameProcessor
  -> SourceBufferStream

MSE 路径的复杂度在 SourceBufferStream：它要维护 append window、timestampOffset、track buffer、keyframe dependency 和 eviction。

5. seek 为什么要找关键帧

Decoder 不能从任意 P/B 帧开始，因为 delta frame 依赖之前的参考帧。Demuxer seek 的目标不是精确的 presentation time，而是目标时间之前最近的可解码点。

Seek(12.5s)
  -> SourceBufferStream::Seek(12.5s)
  -> find keyframe <= 12.5s
  -> discard frames before keyframe
  -> verify dependencies
  -> decoder starts from keyframe

6. DTS 与 PTS

B 帧会让解码顺序和显示顺序不同。Demuxer/stream 需要保留两套时间关系。

DTS order: I1  P4  B2  B3  P7  B5  B6
PTS order: I1  B2  B3  P4  B5  B6  P7
