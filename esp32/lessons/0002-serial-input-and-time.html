让 loop() 同时处理输入和时间

第 2 课 · 预计 25 分钟 · 串口命令实践

本课成果：程序继续每秒心跳，但你可以在监视器里按 1、2、3，立即把周期切换为 250、1000、2500 ms。你将证明等待时间时 ESP32 没有停止处理输入。

1. 先从记忆回答

当前代码为什么使用 millis() 比较，而不是在每次翻转 GPIO2 后直接调用 delay(1000)？先说出一个你认为重要的区别。

2. 把常量变成状态

打开 esp32-test/src/main.cpp。把固定周期：

constexpr uint32_t kIntervalMs = 2500;

改为默认值和可变状态：

constexpr uint32_t kDefaultIntervalMs = 1000;

uint32_t intervalMs = kDefaultIntervalMs;

然后把 loop() 中的 kIntervalMs 改成 intervalMs。

3. 加入命令处理器

在 setup() 前加入：

void handleSerialInput() {
  while (Serial.available() > 0) {
    const char command = static_cast<char>(Serial.read());

    switch (command) {
      case '1': intervalMs = 250; break;
      case '2': intervalMs = 1000; break;
      case '3': intervalMs = 2500; break;
      case '\r':
      case '\n': return;
      default:
        Serial.printf("unknown command: %c\n", command);
        return;
    }

    Serial.printf("interval_ms=%lu\n",
                  static_cast<unsigned long>(intervalMs));
  }
}

在 loop() 第一行调用：

handleSerialInput();

在 setup() 的说明文字后补一行：

Serial.println("Commands: 1=250ms 2=1000ms 3=2500ms");
为什么暂时不用实体按键：你的摄像头板引脚映射尚未确认。随便选择 GPIO 可能与 OV2640 冲突；串口输入可以先学同一个状态机概念。

4. 编译、烧录、验证

cd /home/teng.wang/Code/Teach/esp32/esp32-test
pio run -t upload
pio device monitor

在监视器中依次按 1、3、2。每次都要观察两种证据：

interval_ms=250
heartbeat=... uptime_ms=... gpio2=...

interval_ms=2500
heartbeat=... uptime_ms=... gpio2=...

如果命令能立刻响应，而心跳继续递增，你已经让两个逻辑任务共享同一个 loop()。

5. 小挑战

不复制新代码：增加命令 0，让它恢复默认周期。完成前先预测需要修改哪一处，完成后用串口输出证明预测正确。
