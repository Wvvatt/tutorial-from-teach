让 ESP32 连上网

第 4 课 · 阶段二第 1 课 · 预计 30 分钟 · 真实设备实践

本课成果：ESP32 启动后自动连上你当前局域网，串口打印 IP、信号强度和重连次数；你拔掉路由器/输错密码时，能看到状态从 CONNECTED 跳到 CONNECTION_LOST 再自动回来——而心跳和串口命令全程不被冻结。这一课把第 2 课的非阻塞直觉第一次用在外设上。

1. 先回忆

第 2 课里，你为什么用 millis() 比较而不是 delay(2500)？请用“控制权”和“状态”两个词回答。这一课的每个 Wi-Fi 检查都要遵守同一条：不能把 loop() 锁死。

2. 准备凭据，别提交进仓库

新建文件 esp32-test/src/secrets.h，写入你自己的 SSID 和密码：

#pragma once
#define WIFI_SSID "你的SSID"
#define WIFI_PASS "你的密码"
安全：这是密钥，不要提交进 git。在 esp32-test/.gitignore（没有就新建）里加一行 src/secrets.h。如果仓库已跟踪它，先 git rm --cached src/secrets.h 再忽略。

把凭据单独放在一个文件里，是为了让 main.cpp 可以毫无顾忌地展示和共享，而密码只留在你本机。以后的摄像头帧、HTTP 接口都不需要再碰它。

3. 加状态机骨架

打开 esp32-test/src/main.cpp，在顶部 include 区加：

#include <WiFi.h>
#include "secrets.h"

在现有的匿名命名空间里（和 intervalMs 同一处）加几个 Wi-Fi 状态变量：

constexpr uint32_t kWifiCheckIntervalMs = 500;
uint32_t lastWifiCheckMs = 0;
wl_status_t lastWifiStatus = WL_NO_SHIELD;
uint32_t wifiRetry = 0;
bool wifiIpPrinted = false;

这些就是第 2 课“状态”的延续：lastWifiStatus 记录上次看到的状态，wifiRetry 记录断了几次。

4. 把状态名变成可读文字

在 handleSerialInput() 之前加一个映射函数，把枚举转成名字，免得日志里只看到数字：

static const char* wifiStatusName(wl_status_t s) {
    switch (s) {
    case WL_NO_SHIELD:       return "NO_SHIELD";
    case WL_IDLE_STATUS:     return "IDLE";
    case WL_NO_SSID_AVAIL:   return "NO_SSID";
    case WL_CONNECTED:       return "CONNECTED";
    case WL_CONNECT_FAILED:  return "CONNECT_FAILED";
    case WL_CONNECTION_LOST: return "CONNECTION_LOST";
    case WL_DISCONNECTED:    return "DISCONNECTED";
    default:                 return "?";
    }
}

这串名字的取值来自你本机安装的 libraries/WiFi/src/WiFiType.h，不是文档截图——以本地头文件为准。

5. 一次性启动连接

在 setup() 里、那一堆 Serial.printf 自报家门之后，加一个只跑一次的启动函数：

static void wifiBegin() {
    WiFi.mode(WIFI_STA);
    WiFi.setAutoReconnect(true);
    WiFi.begin(WIFI_SSID, WIFI_PASS);
    lastWifiStatus = WiFi.status();
    Serial.printf("wifi: connecting to %s (start=%s)\n",
                  WIFI_SSID, wifiStatusName(lastWifiStatus));
}

然后在 setup() 末尾调用 wifiBegin();。

关键点：WiFi.begin() 是非阻塞的，它一调用就返回，真正的连接在后台进行。我们靠轮询 WiFi.status() 来观察进度——这和心跳用 millis() 是同一种思路：把控制权留在 loop() 里。

6. 在 loop 里非阻塞地观察

加一个每次 loop() 都会调用的函数，用第 2 课的时间门控：

static void wifiLoop() {
    const uint32_t now = millis();
    if (now - lastWifiCheckMs < kWifiCheckIntervalMs) return;
    lastWifiCheckMs = now;

    const wl_status_t s = WiFi.status();

    if (s == WL_CONNECTED) {
        if (!wifiIpPrinted) {
            Serial.printf("wifi: CONNECTED ip=%s rssi=%d retries=%lu\n",
                          WiFi.localIP().toString().c_str(),
                          static_cast<int>(WiFi.RSSI()),
                          static_cast<unsigned long>(wifiRetry));
            wifiIpPrinted = true;
        }
        return;
    }

    wifiIpPrinted = false;  // 一旦掉线，下次连上要重新打印

    if (s != lastWifiStatus) {
        if (lastWifiStatus == WL_CONNECTED) ++wifiRetry;
        Serial.printf("wifi: %s -> %s retries=%lu\n",
                      wifiStatusName(lastWifiStatus),
                      wifiStatusName(s),
                      static_cast<unsigned long>(wifiRetry));
        lastWifiStatus = s;
    } else {
        Serial.print(".");  // 同一状态停留时，每 500ms 推进一下进度
    }
}

在 loop() 顶部、handleSerialInput(); 之后调用 wifiLoop();。现在 loop() 同时承担三件事：读串口命令、推进 Wi-Fi 观察、维持心跳——且没有一个是阻塞的。

7. 编译、烧录、验证

cd /home/teng.wang/Code/Teach/esp32/esp32-test
pio run -t upload
pio device monitor

正常路径应看到（行号略）：

=== ESP32 test firmware ===
Free heap: ... bytes
wifi: connecting to 你的SSID (start=IDLE)
wifi: IDLE -> CONNECTED ip=192.168.x.x rssi=-52 retries=0
heartbeat=1 uptime_ms=... free_heap=... gpio2=HIGH

心跳继续递增，证明 loop() 没有被 Wi-Fi 冻结。

8. 故意制造一次断线

  1. 在串口监视器里按 1 / 2 / 3 仍能立刻切换周期——验证 Wi-Fi 与命令共存。
  2. 关掉路由器 Wi-Fi 或拔离太远，约 5 秒内查看日志，应看到 CONNECTED -> DISCONNECTED 或 -> CONNECTION_LOST，retries 增加。
  3. 恢复 AP，因为 setAutoReconnect(true)，应看到自动回到 CONNECTED 并重新打印 IP。
  4. 把 secrets.h 里的密码改错一个字符重新烧录，应看到停在 CONNECT_FAILED——这是一类“没起来”的故障，对应第 3 课的三分类。

9. 小挑战

在 handleSerialInput() 里加一个命令 r，调用 WiFi.reconnect();，并立刻清空 wifiIpPrinted 和 lastWifiStatus。预测一下：按 r 之后，你会先看到哪个状态？（提示：强制重连不是立即 WL_CONNECTED）。用日志验证你的预测。

10. 回忆检查

本机 wl_status_t 枚举里，唯一表示“已连接并可用”的是哪一个？为什么我们非要在 loop() 里轮询它，而不能沿用官方示例里的 while (WiFi.status() != WL_CONNECTED) delay(500);？请在答案中同时出现“WL_CONNECTED”和“控制权”。

通过标准

  • ESP32 自动连上当前局域网，串口打印 IP 和 RSSI，心跳不中断。
  • 能说出 WL_CONNECTED 之外至少三个状态的含义。
  • 能解释为什么用 millis() 轮询而非阻塞 while+delay，并指出后者会破坏什么。
  • 能看到一次真实的断线→自动重连，并能从 retries 数字读出发生了什么。
