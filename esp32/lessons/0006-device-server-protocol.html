让 ESP32 主动汇报一次

第 6 课 · 预计 30 分钟 · JSON POST 闭环

本课成果：按串口命令 p，ESP32 向 Mac 的 POST /events 发送一条 JSON；Mac 打印请求体并返回 201 Created，ESP32 串口打印这个状态码。这就是以后上传 JPEG 的协议骨架。

0. 开课门槛

先用 Mac 成功执行一次 curl -i http://ESP32_IP/health。这证明双方同网段且 ESP32 已拿到 IP。

1. 先分清角色

上一课中，浏览器是客户端，ESP32 是服务端。本课角色翻转：ESP32 是客户端，Mac 是服务端。客户端发请求，服务端决定状态码。一次 HTTP 连接中，“客户端/服务端”由谁发起请求决定，不由设备大小决定。

2. 在 Mac 启动最小接收器

新建 server/receiver.py：

from http.server import BaseHTTPRequestHandler, ThreadingHTTPServer

class Handler(BaseHTTPRequestHandler):
    def do_POST(self):
        if self.path != "/events":
            self.send_error(404)
            return

        length = int(self.headers.get("Content-Length", "0"))
        body = self.rfile.read(length)
        print("event:", body.decode("utf-8"))

        response = b'{"accepted":true}\n'
        self.send_response(201)
        self.send_header("Content-Type", "application/json")
        self.send_header("Content-Length", str(len(response)))
        self.end_headers()
        self.wfile.write(response)

ThreadingHTTPServer(("0.0.0.0", 8000), Handler).serve_forever()

运行：

python3 server/receiver.py

另开终端先用 Mac 自测协议：

curl -i -X POST http://127.0.0.1:8000/events \
  -H 'Content-Type: application/json' \
  -d '{"uptime_ms":1234}'

必须先看到 201 Created，再把 ESP32 加入链路。这样出错时知道服务端本身是好的。

3. 找到 Mac 的局域网 IP

ipconfig getifaddr en0

记下结果，例如 192.168.1.10。不要把 127.0.0.1 写进 ESP32：对 ESP32 来说，那表示 ESP32 自己。

4. 加一个主动 POST

在 include 区加入：

#include <HTTPClient.h>

加入函数，把示例 IP 换成你的 Mac IP：

static void postEvent() {
    if (WiFi.status() != WL_CONNECTED) {
        Serial.println("post: skipped, wifi disconnected");
        return;
    }

    HTTPClient http;
    http.begin("http://192.168.1.10:8000/events");
    http.addHeader("Content-Type", "application/json");

    String body = "{\"uptime_ms\":" + String(millis())
                + ",\"free_heap\":" + String(ESP.getFreeHeap()) + "}";
    const int status = http.POST(body);
    Serial.printf("post: status=%d body=%s\n", status, body.c_str());
    http.end();
}

在串口命令的 switch 中加入：

case 'p':
    postEvent();
    return;

5. 运行反馈回路

pio run -t upload
pio device monitor

按 p。两端应分别看到：

# ESP32
post: status=201 body={"uptime_ms":...,"free_heap":...}

# Mac receiver
event: {"uptime_ms":...,"free_heap":...}

再把路径临时改成 /wrong，预测并验证返回 404。恢复后，把接收器停掉再按 p，观察负数错误码。这三种证据分别表示：协议成功、路径错误、服务器不可达。

6. 本课的限制

HTTPClient::POST() 会等待网络结果。本课只允许由按键命令触发，用来学习协议边界。后续定时拍照上传时，我们会加入超时、发送状态和失败策略，不能让它无边界地占用控制权。

7. 回忆检查

为什么 ESP32 不能把 127.0.0.1 当作 Mac？为什么收到 404 时不能把上传记为成功？请在答案中使用“本机”“路径”“状态码”。

通过标准

  • Mac 自测 POST 返回 201。
  • ESP32 按 p 后 Mac 收到同一条 JSON。
  • 能从 201、404、负数错误码区分三类结果。
  • 发送期间和发送后能继续看到心跳。
