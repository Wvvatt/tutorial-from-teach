抓取第一张 JPEG

第 8 课 · 预计 30 分钟 · 串口可见结果

本课成果：启动时识别 OV2640，按串口命令 c 抓取一帧 QVGA JPEG，打印宽高、字节数、格式和 JPEG 首尾标记；连续抓三次都能成功且心跳继续。

1. 先回忆内存所有权

摄像头驱动拥有一组帧缓冲。esp_camera_fb_get() 暂时把一个缓冲借给你；处理完必须调用 esp_camera_fb_return() 归还。这里不是普通的“读取一个值”，而是借用一块可能位于 PSRAM 的大内存。

驱动缓冲池 → fb_get() → 你的代码 → fb_return() → 驱动缓冲池

2. 引入驱动并保存初始化状态

在 include 区加入：

#include <esp_camera.h>

在匿名命名空间加入：

bool cameraReady = false;

3. 填入已验证的 AI Thinker 兼容引脚

加入初始化函数：

static bool cameraBegin() {
    camera_config_t config{};
    config.ledc_channel = LEDC_CHANNEL_0;
    config.ledc_timer = LEDC_TIMER_0;

    config.pin_d0 = 5;
    config.pin_d1 = 18;
    config.pin_d2 = 19;
    config.pin_d3 = 21;
    config.pin_d4 = 36;
    config.pin_d5 = 39;
    config.pin_d6 = 34;
    config.pin_d7 = 35;
    config.pin_xclk = 0;
    config.pin_pclk = 22;
    config.pin_vsync = 25;
    config.pin_href = 23;
    config.pin_sccb_sda = 26;
    config.pin_sccb_scl = 27;
    config.pin_pwdn = 32;
    config.pin_reset = -1;

    config.xclk_freq_hz = 20000000;
    config.pixel_format = PIXFORMAT_JPEG;
    config.frame_size = FRAMESIZE_QVGA;
    config.jpeg_quality = 12;
    config.fb_count = 1;
    config.grab_mode = CAMERA_GRAB_WHEN_EMPTY;
    config.fb_location = CAMERA_FB_IN_PSRAM;

    const esp_err_t err = esp_camera_init(&config);
    if (err != ESP_OK) {
        Serial.printf("camera: init failed err=0x%x\n", err);
        return false;
    }

    sensor_t* sensor = esp_camera_sensor_get();
    Serial.printf("camera: ready pid=0x%04x psram=%s\n",
                  sensor->id.PID, psramFound() ? "yes" : "no");
    return true;
}
为什么先用 QVGA + 单缓冲：我们已经确认 PSRAM，但第一次实验只验证传感器、GPIO、DMA 和缓冲生命周期。先减少变量，稳定后再提高分辨率或使用双缓冲。

4. 抓取并验证一帧

加入函数：

static void captureOnce() {
    if (!cameraReady) {
        Serial.println("camera: not ready");
        return;
    }

    camera_fb_t* fb = esp_camera_fb_get();
    if (fb == nullptr) {
        Serial.println("camera: capture failed");
        return;
    }

    const bool jpegStart = fb->len >= 2
        && fb->buf[0] == 0xFF && fb->buf[1] == 0xD8;
    const bool jpegEnd = fb->len >= 2
        && fb->buf[fb->len - 2] == 0xFF
        && fb->buf[fb->len - 1] == 0xD9;

    Serial.printf("camera: %zux%zu bytes=%zu format=%d jpeg=%s\n",
                  fb->width, fb->height, fb->len,
                  static_cast<int>(fb->format),
                  jpegStart && jpegEnd ? "valid" : "invalid");

    esp_camera_fb_return(fb);
}

FF D8 是 JPEG 的 SOI（Start of Image），FF D9 是 EOI（End of Image）。这不能证明画面好看，但能证明缓冲区像一份完整 JPEG。

5. 接入 setup 和串口命令

在 setup() 中、打印 PSRAM 之后调用：

cameraReady = cameraBegin();

在串口命令 switch 中加入：

case 'c':
    captureOnce();
    return;

如果编译器提示函数尚未声明，把 cameraBegin() 和 captureOnce() 放在 handleSerialInput() 之前。

6. 编译、烧录、验证

pio run -t upload
pio device monitor

启动应看到：

[RAM: external PSRAM]
found=yes total=4192123 bytes free=4192123 bytes
camera: ready pid=0x2642 psram=yes

按三次 c，期望类似：

camera: 320x240 bytes=12345 format=4 jpeg=valid
camera: 320x240 bytes=11980 format=4 jpeg=valid
camera: 320x240 bytes=12117 format=4 jpeg=valid

每帧字节数不同是正常的：JPEG 是有损压缩，画面内容不同会产生不同大小。三次之间心跳必须继续。

7. 故障签名

  • init failed：先记录完整十六进制错误码，不要盲改 GPIO。
  • capture failed：检查供电、排线和 PSRAM；确认初始化确实成功。
  • jpeg=invalid：记录长度及首尾字节，检查是否真是 PIXFORMAT_JPEG。
  • 第一次成功、后面失败：优先检查是否遗漏 esp_camera_fb_return(fb)。
