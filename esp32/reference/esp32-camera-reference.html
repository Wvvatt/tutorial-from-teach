ESP32 + OV2640 速查

这张表只保留课程中反复会用到的概念。先记住数据流，再记 API 名称：摄像头产生一帧 JPEG，ESP32 拿到帧缓冲，网络层负责把字节交给浏览器或 server。

一帧照片的生命周期

esp_camera_init(config)
        ↓
esp_camera_fb_get()  →  camera_fb_t { buf, len, width, height, format }
        ↓
HTTP response / HTTP POST
        ↓
esp_camera_fb_return(fb)

拿到帧后必须归还帧缓冲；否则下一次抓拍可能因为缓冲耗尽失败。JPEG 是网络传输的起点，RGB565 更适合像素处理但通常占用更多内存。

OV2640 与板型

概念课程中的含义
传感器型号OV2640 是 2 MP 彩色摄像头，支持 JPEG 等输出格式。
板型引脚ESP32 的 D0-D7、XCLK、PCLK、VSYNC、HREF、SCCB 和电源控制脚由开发板布线决定，不能只凭传感器型号猜。
PSRAM高分辨率或多帧缓冲通常需要 PSRAM；初始化前用 psramFound() 判断并降低配置作为 fallback。
帧尺寸先用 QVGA/VGA 验证链路，再提高到 UXGA；先保证稳定，再追求画质。

最小配置骨架

camera_config_t config{};
config.pixel_format = PIXFORMAT_JPEG;
config.frame_size = FRAMESIZE_QVGA;
config.jpeg_quality = 12;
config.fb_count = psramFound() ? 2 : 1;
config.fb_location = psramFound() ? CAMERA_FB_IN_PSRAM : CAMERA_FB_IN_DRAM;
esp_err_t err = esp_camera_init(&config);
不要只按传感器型号抄引脚：这组 GPIO 来自你这块板子的照片、AI Thinker 对应配置和实机 PSRAM 验证；换另一种 ESP32-CAM 板型时要重新确认。
