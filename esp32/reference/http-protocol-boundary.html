HTTP 协议边界速查

对应 第 6 课。

一次交换

ESP32                         Mac
  | POST /events HTTP/1.1      |
  | Content-Type: application/json
  | {"uptime_ms":1234}         |
  |--------------------------->|
  | HTTP/1.1 201 Created       |
  | {"accepted":true}          |
  |<---------------------------|

边界四要素

要素问题
方法GET 读取，POST 提交一份数据
路径/events 是双方约定的资源名
媒体类型application/json 告诉接收方如何解释请求体
状态码2xx 成功；4xx 请求有问题；5xx 服务端失败

ESP32 客户端骨架

HTTPClient http;
http.begin("http://192.168.1.10:8000/events");
http.addHeader("Content-Type", "application/json");
int status = http.POST("{\"uptime_ms\":1234}");
http.end();

原始 JPEG 上传

camera_fb_t* fb = esp_camera_fb_get();

HTTPClient http;
http.setConnectTimeout(2000);
http.setTimeout(5000);
http.begin("http://MAC_IP:8000/images");
http.addHeader("Content-Type", "image/jpeg");
int status = http.POST(fb->buf, fb->len);

esp_camera_fb_return(fb);
http.end();
响应客户端含义
201服务器接受了完整 JPEG
4xx路径、媒体类型、大小或 JPEG 内容不符合协议
5xx服务端处理失败
负数连接、DNS、超时等客户端网络错误，不是 HTTP 状态码

JPEG 已经是二进制格式，直接作为请求正文发送；不要先转 Base64。POST() 返回后网络层不再读取原帧，调用者应尽快把它归还摄像头缓冲池。

状态码是协议证据，不是装饰。只有拿到预期的 2xx，设备才应该把本次发送记作成功。
