HTTP 协议边界速查

对应 第 6 课。

一次交换

ESP32                         Mac
  | POST /events HTTP/1.1      |
  | Content-Type: application/json
  | {"uptime_ms":1234}         |
  |--------------------------->|
  | HTTP/1.1 201 Created       |
  | {"accepted":true}          |
  |<---------------------------|

边界四要素

要素问题
方法GET 读取，POST 提交一份数据
路径/events 是双方约定的资源名
媒体类型application/json 告诉接收方如何解释请求体
状态码2xx 成功；4xx 请求有问题；5xx 服务端失败

ESP32 客户端骨架

HTTPClient http;
http.begin("http://192.168.1.10:8000/events");
http.addHeader("Content-Type", "application/json");
int status = http.POST("{\"uptime_ms\":1234}");
http.end();
状态码是协议证据，不是装饰。只有拿到预期的 2xx，设备才应该把本次发送记作成功。

主资料：Espressif 官方 HTTPClient 示例；Python 官方 http.server 文档。
