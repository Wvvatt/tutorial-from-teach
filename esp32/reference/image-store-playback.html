图片存储与播放速查

对应 第 12 课。

原子发布

POST body
  → 验证 JPEG
  → 写 .tmp
  → flush + fsync + close
  → os.replace(.tmp, .jpg)
  → 查询 API 才能列出
公开列表只匹配 *.jpg。读取者看不到正在写入的临时文件；同文件系统内的 os.replace() 提供原子名称切换。

服务边界

接口职责
POST /images验证、保存并返回 201 与公开 URL
GET /api/images返回最近 N 张的有序元数据
GET /images/name.jpg只提供安全文件名对应的不可变 JPEG
GET /提供播放器页面

文件名

20260717T081530.123456Z-a1b2c3.jpg
└────── UTC 接收时间 ──────┘  └唯一后缀┘

按文件名排序即可得到接收时间顺序。它不是曝光时间；若需要真实拍摄时间，设备时钟必须同步并通过协议传递。

仍需设计

容量上限、删除策略、认证、上传退避、结构化指标和设备端有限缓存属于可靠性层，不由“能保存”自动解决。
