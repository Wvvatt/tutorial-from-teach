第 3 课 · 非阻塞 loop

内存与故障反馈速查

ESP32 的调试不是猜“哪里坏了”，而是读懂三类证据：当前还剩多少堆内存、程序是否在阻塞、以及它是不是在反复重启。

常用打印

Serial.printf("Free heap: %lu bytes\n",
              static_cast<unsigned long>(ESP.getFreeHeap()));
void* probe = malloc(1024 * 1024);
if (probe == nullptr) {
  Serial.println("malloc failed");
}

三种信号

信号通常说明该看什么
持续心跳 + heap 数字程序还在跑数值是否下降、是否有任务堆积
输出停住卡在阻塞调用delay、等待 IO、死循环
启动日志重复出现崩溃后重启最后一条日志、复位原因、异常回溯

三个判断

看字节数，不看感觉；看日志序列，不看单行；先分清“阻塞”和“重启”，再去查 Wi-Fi、摄像头或上传代码。
