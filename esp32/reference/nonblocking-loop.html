非阻塞 loop() 速查

一个好的主循环每次只做一点工作，然后尽快返回。串口、Wi-Fi、摄像头和上传任务才能轮流获得处理机会。

周期任务

uint32_t lastRunMs = 0;
uint32_t intervalMs = 1000;

void loop() {
  const uint32_t now = millis();
  if (now - lastRunMs >= intervalMs) {
    lastRunMs = now;
    runPeriodicTask();
  }

  handleInput();
}

串口输入

void handleInput() {
  while (Serial.available() > 0) {
    const char command = static_cast<char>(Serial.read());
    handleCommand(command);
  }
}

判断表

写法影响适用场景
delay(2000)当前任务停住约 2 秒短暂启动等待、一次性小实验
millis() 比较等待期间仍能处理其他任务心跳、重连、拍照、周期上传
while (available())清空已经到达的输入字节命令、传感器报文、串口协议

三个规则

状态放在循环外；到时间才执行周期任务；没有工作时尽快把控制权交回系统。
