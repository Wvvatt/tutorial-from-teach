ESP32-CAM 运行手册

完成课程后用于启动、检查和恢复系统。

启动

cd /Users/watt/Code/Teach/esp32/esp32-test
python3 server/server.py

# 需要更新固件时
pio run -t upload
pio device monitor

健康检查

curl http://192.168.1.195/health
curl http://127.0.0.1:8000/api/status
curl http://127.0.0.1:8000/api/images
open http://127.0.0.1:8000/

结果解释

证据含义
upload_last_status=201最近图片已被服务器接受
负数状态连接/超时错误；设备会退避
404/413/415/422协议错误；自动上传停用
upload_consecutive_failures=0当前不在连续失败中
stored_images ≤ 120服务端保留边界正常

恢复顺序

1. 确认 Mac server 正在监听 8000
2. 检查 ESP32 /health 与 Wi‑Fi IP
3. 检查 upload_last_status
4. 协议错误：修复 URL/正文，按 u 重新启用
5. 暂时故障：等待退避，不要快速重启设备
6. 摄像头失败：检查初始化错误码、供电、排线、PSRAM

安全停止

Python server 使用 Ctrl+C 正常停止。设备保持运行时会进入有限退避；服务恢复后无需重启 ESP32。不要在图片临时文件写入时强制删除整个目录。

先读取证据，再改变状态。一次只改一个边界：设备、网络、协议或存储。
