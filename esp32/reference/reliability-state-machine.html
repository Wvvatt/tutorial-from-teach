上传可靠性状态机速查

对应 第 13 课。

结果到策略

结果动作
201成功；连续失败归零，恢复正常间隔
负数 / 5xx / 408 / 429 / 抓拍失败暂时故障；10→20→40→60 秒退避
404/413/415/422协议错误；停止自动上传，等待修复
Wi‑Fi 未连接不抓拍、不计失败；等待连接状态机

有界资源

设备待上传帧：0 张（失败即丢弃并归还）
单次连接超时：2 秒
单次请求超时：5 秒
最大重试间隔：60 秒
服务端图片上限：120 张
公开列表上限：60 张
边界必须体现在状态和代码中。“应该不会无限增长”不是可靠性保证。

最小可观测字段

upload_enabled
upload_successes
upload_failures
upload_consecutive_failures
upload_last_status
upload_next_in_ms
stored_images
stored_bytes
max_stored_images

故障注入

至少验证：服务停止、服务恢复、错误路径、磁盘保留上限。没有故障证据的重试代码，只是未经检验的愿望。

主资料：Espressif 官方错误处理指南、FreeRTOS 任务延迟文档和 Python Lock 文档。
