运行时配置速查

安全更新顺序：读取输入 → 完整解析 → 检查范围 → 一次性提交状态 → 返回新状态。

接口约定

GET  /config
200 {"interval_ms":30000}

POST /config?interval_ms=30000
200 {"interval_ms":30000}

缺失、非数字、范围外
400 text/plain

状态层次

  • 编译期常量：固件功能边界，例如允许的最小值和最大值。
  • 运行时状态：当前上传间隔，重启会丢失。
  • 持久化配置：写入 NVS/Flash，重启后恢复；需要考虑默认值、版本和写入寿命。

并发判断

不要因为变量“全局”就机械加锁。先画出访问者：当前 handleClient() 和 uploadLoop() 都从同一个 loop() 顺序调用，没有同时执行。若以后把配置 handler 或上传放入独立 FreeRTOS task，共享状态的同步假设就改变了。

输入校验

String::toInt() 无法清楚地区分非法文本和数值 0。strtoul() 配合结束指针可以确认整个字符串都被解析，再做上下界检查。
