Wi-Fi 连接速查

对应 第 4 课。API 来自已安装的 arduino-esp32 WiFiSTA.h / WiFiType.h。

wl_status_t 枚举

来源：libraries/WiFi/src/WiFiType.h

值名称含义
255WL_NO_SHIELDWi-Fi 还没启动/不可用（兼容旧库）
0WL_IDLE_STATUS空闲，正在切换状态
1WL_NO_SSID_AVAIL找不到该 SSID
2WL_SCAN_COMPLETED扫描完成
3WL_CONNECTED已连接并拿到 IP（唯一“成功”）
4WL_CONNECT_FAILED连接失败（密码错等）
5WL_CONNECTION_LOST连上后断开
6WL_DISCONNECTED已断开

核心调用

WiFi.mode(WIFI_STA);          // 站点模式（必须在 begin 前）
WiFi.setAutoReconnect(true);  // 断线时由驱动自动重连
WiFi.begin(ssid, password);   // 发起连接（非阻塞，立即返回）

WiFi.status();                // 返回 wl_status_t，必须轮询
WiFi.localIP();               // IPAddress，仅 status==WL_CONNECTED 时有意义
WiFi.RSSI();                  // int8_t，信号强度，正值不可能
WiFi.reconnect();             // 手动强制重连（配合命令调试）

非阻塞轮询模式（替换官方的 while+delay）

uint32_t lastCheck = 0;
wl_status_t last = WL_NO_SHIELD;

void wifiLoop() {
  if (millis() - lastCheck < 500) return;
  lastCheck = millis();
  wl_status_t s = WiFi.status();
  if (s != last) {
    Serial.printf("wifi: %s -> %s\n", name(last), name(s));
    last = s;
  }
}

不要用 while (WiFi.status() != WL_CONNECTED) { delay(500); }：它会冻结 loop()，与后续 HTTP 服务和抓帧任务冲突。

诊断口诀

串口现象先查
一直停在 IDLE / DISCONNECTEDSSID 拼写、5GHz vs 2.4GHz、密码
NO_SSID_AVAIL路由器是否在 2.4GHz、信号是否够强
CONNECT_FAILED 反复密码、认证类型（WPA2/WPA3）
连上后 CONNECTION_LOST 反复电源、距离、AP 漫游

主资料：Arduino-ESP32 WiFi API。枚举以本机安装版本 WiFiType.h 为准。
