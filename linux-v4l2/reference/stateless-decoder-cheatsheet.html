Stateless Decoder 速查表

初始化序列

  1. S_FMT(OUTPUT) — 设置 coded format（如 V4L2_PIX_FMT_H264_SLICE）
  2. S_EXT_CTRLS — 设置初始化参数（profile/level 等）
  3. G_FMT(CAPTURE) — 获取推荐 CAPTURE 格式
  4. [S_FMT(CAPTURE)] — 可选：选择不同 raw format
  5. REQBUFS(OUTPUT)
  6. REQBUFS(CAPTURE)
  7. MEDIA_IOC_REQUEST_ALLOC — 分配 request（通常每 OUTPUT buffer 一个）
  8. STREAMON(OUTPUT) + STREAMON(CAPTURE)

每帧解码循环（核心伪代码）

// 对每一帧：
struct v4l2_buffer out = {
    .type = OUTPUT_MPLANE, .index = idx,
    .bytesused  = frame_size,
    .flags      = V4L2_BUF_FLAG_REQUEST_FD,
    .request_fd = req_fd,           // 关联到 request
    .timestamp  = frame_ts,          // 唯一 ID，会传播到 CAPTURE
};
ioctl(fd, VIDIOC_QBUF, &out);

struct v4l2_ext_controls ctrls = {
    .which      = V4L2_CTRL_WHICH_REQUEST_VAL,
    .request_fd = req_fd,
    .count      = n,
    .controls   = ctrl_array,       // SPS/PPS/SliceParams/DecodeParams/...
};
ioctl(fd, VIDIOC_S_EXT_CTRLS, &ctrls);

ioctl(req_fd, MEDIA_REQUEST_IOC_QUEUE);   // 原子提交

// 取结果（CAPTURE buffer 不在 request 中，独立出队）
ioctl(fd, VIDIOC_DQBUF, &capture_buf);

约束：至少 1 个 OUTPUT buffer 才能 QUEUE（否则 -ENOENT）。只能有 1 个 OUTPUT buffer（多个 -EINVAL）。CAPTURE buffer 不能进入 request。

DRC（Dynamic Resolution Change）— Stateless 版本

  1. 如有 held CAPTURE buffer → V4L2_DEC_CMD_FLUSH
  2. 等待所有 pending request 完成并 DQBUF
  3. STREAMOFF(OUTPUT) + STREAMOFF(CAPTURE)
  4. REQBUFS(CAPTURE, count=0) — 释放 CAPTURE buffer
  5. 用新分辨率重新执行初始化序列（OUTPUT buffer 可以保留不重新分配）
