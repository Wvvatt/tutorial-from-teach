vb2_ops 语义

回调何时发生常见职责
queue_setupREQBUFS/CREATE_BUFS决定 buffer 数量、plane 数量、每 plane size
buf_preparebuffer 从用户交给内核时验证 bytesused、plane size、payload 合法性
buf_queuebuffer 进入 driver queue 时通常调用 v4l2_m2m_buf_queue()
start_streaming队列进入 streaming打开 clock/power、初始化硬件队列、清状态
stop_streamingSTREAMOFF 或 close停止 DMA/硬件，归还所有未完成 buffer
buf_finishDQBUF 前可做 cache sync、metadata 修正

m2m_ops 语义

回调语义注意点
device_run开始一次 transaction通常异步返回；完成时必须调用 job finish
job_ready判断当前 context 是否真的可运行不可睡眠；用于额外条件，如 header 已解析、buffer 数量足够
job_abort请求尽快中止当前 transaction安全停止后仍需按完成路径释放 job
基础调度条件：至少一个 source/OUTPUT buffer、至少一个 destination/CAPTURE buffer、streaming 已开启。driver 的 job_ready 可以加更严格条件。

一次 job 的完成路径

interrupt / firmware callback
  -> identify src/dst buffers
  -> set bytesused / timestamp / flags
  -> mark OUTPUT done
  -> mark CAPTURE done
  -> v4l2_m2m_job_finish()

stateless multi-slice drivers:
  -> v4l2_m2m_buf_done_and_job_finish()
     handles held CAPTURE buffers
