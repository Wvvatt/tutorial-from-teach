vicodec 源码阅读地图

目标：从设备注册一路追到一个 mem2mem job 的完成。源码入口是 Linux 内核的 vicodec-core.c。

主路径

vicodec_probe
  → register_instance
    → v4l2_m2m_init(&m2m_ops)
    → video_register_device
  → vicodec_open
    → v4l2_m2m_ctx_init(..., queue_init)
      → OUTPUT/CAPTURE vb2_queue
  → QBUF
    → vicodec_buf_queue
      → v4l2_m2m_buf_queue
        → v4l2_m2m_try_schedule
          → job_ready
            → device_run
              → buffer_done
              → v4l2_m2m_job_finish

容易误读的三点

QBUF ≠ running：QBUF 只把 buffer 放入 ready queue；job_ready 仍可因 header 不完整、source change 或 Request 缺失返回 0。
DQBUF ≠ processing：DQBUF 只是消费已经被 buffer_done 标记的 buffer。
ctx ≠ dev：vicodec_dev 是设备级对象；vicodec_ctx 是一个 open 的独立状态。
